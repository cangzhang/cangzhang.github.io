Ubuntu11.10编译Android4.0

原文地址：http://www.just-for-you.org/blog/phone/phone-document/2642.html

PS:

ICS的官方说明是在10.04上进行的，但是，想必很多人都在用11.10吧，当然随之而来的就是各种停止、各种报错，这里我写一些我个人编译的一些经验。

————————————————————————————————–

关于源码下载之类的，我想我无需多做说明了，需要的就去看下我写过的帖子。

Android开发系统安装

Android开发环境配置

Android编译环境搭配

这里额外说一下的是，对于依赖文件的安装，务必参考我写过的或者google官方的说明，当然，你也可以试着进行编译，然后看看哪里出错，接着缺少什么安装什么就好，但是这样的，就需要重复很多次，所以我再复制一下吧：

sudo apt-get update
sudo apt-get install git-core gnupg flex bison gperf build-essential zip curl zlib1g-dev libc6-dev lib32ncurses5-dev ia32-libs
x11proto-core-dev libx11-dev lib32z1-dev libgl1-mesa-dev g++-multilib mingw32 tofrodos python-markdown libxml2-utils
————————————————————————————————–
这里不得不说的另一个问题，Ubuntu 11.10默认安装的是GCC 4.6).我们需要换回GCC 4.4来进行编译。

帖子我也写过了，需要的话，建议点击下面的链接自己看看怎么解决这个问题。

【Thonatos分享】Android编译中GCC版本问题

还有一个问题是很多人老是忘记配置环境变量，这里建议你不仅仅要把jdk、repo等加入环境变量，更需要把适当的工具加入环境变量，

如果是需要运行模拟器，建议你继续阅读我这几天发的另一篇文章：

【Thonatos分享】模拟器中运行编译好的Android

 

这里补充说明一下环境变量的问题：
1.你可以利用gedit /etc/environment 来配置全局变量，但是有时候在以root权限执行时，会有一些问题（不在本文讨论范围内。）
2.你可以利用gedit ~/.bashrc 来配置个人变量（用户变量），既然说了是用户变量，你应该懂，它仅仅适用于当前用户，root的用户变量的绝对路径位置是/root/.bashrc，如果你不是以root用户运行，这个变量实际就是无效的，更详细的，建议你去阅读ubuntu的相关说明文档。

————————————————————————————————–
这些都做好之后，当然就该试着运行一下了，

cd /home/android //这个目录是你自己设置的，要对应你自己的源码目录
lunch full-eng //选择编译的版本

make -j4 //对于后面的参与，是指的你的cpu相关的参数，与编译同时使用的任务数，N为其整数倍（2个CPU，每个CPU4个核心，每核心2线程），最快的构建是在j16至j32之间，这里的参数主要影响的是编译时间。
解释说明：
user 适合用于生产的机会有限;
userdebug 用于调试的首选像“用户”，具有根访问权限和调试权限;
eng 与其他调试工具的开发配置
————————————————————————————————–
大体就是这样了，可能机器不同，遇到的问题也不同吧，根据问题查找解决方案吧。
