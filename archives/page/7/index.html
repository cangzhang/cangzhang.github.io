Ubuntu11.10编译Android4.0

原文地址：http://www.just-for-you.org/blog/phone/phone-document/2642.html

PS:

ICS的官方说明是在10.04上进行的，但是，想必很多人都在用11.10吧，当然随之而来的就是各种停止、各种报错，这里我写一些我个人编译的一些经验。

————————————————————————————————–

关于源码下载之类的，我想我无需多做说明了，需要的就去看下我写过的帖子。

Android开发系统安装

Android开发环境配置

Android编译环境搭配

这里额外说一下的是，对于依赖文件的安装，务必参考我写过的或者google官方的说明，当然，你也可以试着进行编译，然后看看哪里出错，接着缺少什么安装什么就好，但是这样的，就需要重复很多次，所以我再复制一下吧：

sudo apt-get update
sudo apt-get install git-core gnupg flex bison gperf build-essential zip curl zlib1g-dev libc6-dev lib32ncurses5-dev ia32-libs
x11proto-core-dev libx11-dev lib32z1-dev libgl1-mesa-dev g++-multilib mingw32 tofrodos python-markdown libxml2-utils
————————————————————————————————–
这里不得不说的另一个问题，Ubuntu 11.10默认安装的是GCC 4.6).我们需要换回GCC 4.4来进行编译。

帖子我也写过了，需要的话，建议点击下面的链接自己看看怎么解决这个问题。

【Thonatos分享】Android编译中GCC版本问题

还有一个问题是很多人老是忘记配置环境变量，这里建议你不仅仅要把jdk、repo等加入环境变量，更需要把适当的工具加入环境变量，

如果是需要运行模拟器，建议你继续阅读我这几天发的另一篇文章：

【Thonatos分享】模拟器中运行编译好的Android

 

这里补充说明一下环境变量的问题：
1.你可以利用gedit /etc/environment 来配置全局变量，但是有时候在以root权限执行时，会有一些问题（不在本文讨论范围内。）
2.你可以利用gedit ~/.bashrc 来配置个人变量（用户变量），既然说了是用户变量，你应该懂，它仅仅适用于当前用户，root的用户变量的绝对路径位置是/root/.bashrc，如果你不是以root用户运行，这个变量实际就是无效的，更详细的，建议你去阅读ubuntu的相关说明文档。

————————————————————————————————–
这些都做好之后，当然就该试着运行一下了，

cd /home/android //这个目录是你自己设置的，要对应你自己的源码目录
lunch full-eng //选择编译的版本

make -j4 //对于后面的参与，是指的你的cpu相关的参数，与编译同时使用的任务数，N为其整数倍（2个CPU，每个CPU4个核心，每核心2线程），最快的构建是在j16至j32之间，这里的参数主要影响的是编译时间。
解释说明：
user 适合用于生产的机会有限;
userdebug 用于调试的首选像“用户”，具有根访问权限和调试权限;
eng 与其他调试工具的开发配置
————————————————————————————————–
大体就是这样了，可能机器不同，遇到的问题也不同吧，根据问题查找解决方案吧。

开发环境配置——源码同步

原文地址：http://www.just-for-you.org/blog/phone/phone-document/2606.html

注意：如非特殊说明，前几节的都是先执行“sudo -i”，在Root权限下执行。

安装Repo：

放置Repo目录：

mkdir ~/bin //首先建立目录： PATH=~/bin:$PATH //设置环境变量

为了方便起见，我是选择直接把Repo放进了/bin目录，因此要是像我一样图方便，可以省略这一步。

下载Repo:

curl https://dl-ssl.google.com/dl/googlesource/git-repo/repo &gt; ~/bin/repo //下载Repo到bin目录 chmod a+x ~/bin/repo //给予运行权限

如果是和我一样偷懒的话，这里的代码应该是这样：
$ curl https://dl-ssl.google.com/dl/googlesource/git-repo/repo &gt; /bin/repo //直接放进/bin目录 $ chmod a+x /bin/repo //给repo运行权限

设置Repo:

mkdir WORKING_DIRECTORY //建立一个工作目录： cd WORKING_DIRECTORY //切换到该目录

本人的做法是：
$ cd /home //先切换到/home目录 $ mkdir android //然后建立android工作目录 $ cd android //然后切换到该目录

下载GIT文件，配置repo：

repo init -u https://android.googlesource.com/platform/manifest //这一步设置工作目录中的repo源 repo init -u https://android.googlesource.com/platform/manifest -b android-4.0.1_r1 //选择下载的版本是android4.0
这里应该会问你什么用户名啊之类的，建议填写你的个人ID，另外邮箱最好是一个真实的google账户的名称，问题应该不大吧。

开始同步源码：
repo sync //开始同步源码

Setting up ccache

You can optionally tell the build to use the ccache compilation tool. Ccache acts as a compiler cache that can be used to speed-up rebuilds. This works very well if you do “make clean” often, or if you frequently switch between different build products.

Put the following in your .bashrc or equivalent.

export USE_CCACHE=1

By default the cache will be stored in ~/.ccache. If your home directory is on NFS or some other non-local filesystem, you will want to specify the directory in your .bashrc as well.

export CCACHE_DIR=

The suggested cache size is 50-100GB. You will need to run the following command once you have downloaded the source code.

prebuilt/linux-x86/ccache/ccache -M 50G

This setting is stored in the CCACHE_DIR and is persistent.

Android开发环境配置——Ubuntu下JAVA安装及环境变量配置

原文地址：http://www.just-for-you.org/blog/phone/phone-document/2506.html

一，下载JAVA-JDK

由于咱们需要配置的是编译环境，所以需要使用相应的JDK版本，这里按照Open Source Project的要求，安装JDK6.0

  • JDK1.7官方下载
  • Java SE 6 Update 30官方下载

Java SE 6 Update 30这个版本经过了多次更新，算是比较稳定的版本了，我们这里就安装这个版本。

点击下载JDK（务必选择JDK，JRE是运行环境，不是编译环境，这个自己去百度哈～），选择bin文件哦～

因为我是按张的64位Ubuntu，所以我选择的jdk-6u30-linux-x64.bin

等待下载完成之后，开始准备安装喽！

二，安装JAVA-JDK

下面就是一路的命令行输入了：

sudo cp jdk-6u30-linux-x64.bin /usr/lib/jvm

复制bin文件至/usr/lib/jvm目录

cd /usr/lib/jvm

切换至该目录

ls -a

查看文件，此时应该会看到jdk-6u30-linux-x64.bin文件了

sudo -i

为了方便起见，我偷偷懒吧，直接切换到root账户了。

chmod a+x jdk-6u30-linux-x64.bin

给予bin文件运行权限

./jdk-6u30-linux-x64.bin

运行该Bin文件，下面会要求回车什么的，看到DONE表示完成。为了保险起见，再键入一次命令

ls -a

这时候应该可以看到多了一个目录吧？就是那个jdk.1.6.0_30出现了吧？这就安装成功了，bin文件删了吧（咱还有备份不是？）

rm -rf jdk-6u30-linux-x64.bin

删除该文件（因为主目录下面还有一个，这里删除了吧）

 

三，配置JAVA环境

据我所知，ubuntu下面的vi是神器一般的存在，但是神器是神用的，一般人用不来——至少我不喜欢！

所以我换用其他编辑器处理。

gedit /etc/environment

然后就出来了环境变量的设置，大家开始复制粘贴吧，下面是我的配置：

PATH="/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games:/usr/lib/jvm/jdk1.6.0_30/bin"

CLASSPATH=”/usr/lib/jvm/jdk1.6.0_30/lib”

JAVA_HOME="/usr/lib/jvm/jdk1.6.0_30"

然后保存文件（保存之后退出即可）此时再输入以下命令是配置生效

source /etcenvironment

使配置生效

但是，有个疑问，我们是否安装成功了呢？

java -version

看到什么？不是JDK1.6.30？呵呵，必须的，还需用继续配置，因为Ubuntu默认是存在Java的。

update-alternatives --install /usr/bin/java java /usr/lib/jvm/jdk1.6.0_30/bin/java 300

update-alternatives --install /usr/bin/javac javac /usr/lib/jvm/jdk1.6.0_30/bin/javac 300

以上是替换默认的java为我们安装的版本

有人要问为什么我没有输入sudo吧？呵呵，还记得我之前输入了“sudo -i”吧？我现在还在使用root权限，所以。。。

update-alternatives --config java

之前的300指的是手动配置，现在是不是要你选择了？恩，输入咱们安装的，然后回车吧

嗯，要不要试验一下？好了，下面输入：

java -version

看到了吧？1.6.0_30配置成功。。。
